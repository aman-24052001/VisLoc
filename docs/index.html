Parameter sandbox

Tune the fusion engine itself, live - this is the same UKF math as the simulation above, ported to JavaScript and re-run on every change. The camera/odometry data comes from a few precomputed noise presets (re-running ORB matching and optical flow live in-browser isn't feasible without porting OpenCV to JS) - switching presets swaps that underlying data, while every slider below recomputes the actual filter in real time.

Results

Standard scenario: 200-frame loop, ±2° camera yaw, σ=1.5px position noise, absolute fix attempted every 10 frames.

Fusion's benefit is scenario-dependent, not universal — it helps most when raw odometry drift is substantial relative to the localizer's own measurement noise (~4px). When VIO is already that accurate on its own, there's little room left for fusion to improve on it. Full validation, including a known open limitation around sharp direction-reversal points, is documented in the repository README.
